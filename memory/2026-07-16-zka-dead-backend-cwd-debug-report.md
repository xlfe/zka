# DEBUG REPORT

- **Symptom:** `zka workspace attach` briefly opened and then closed Kitty, later reporting a missing managed socket. New Kitty tabs/windows created through a `*_with_cwd` action started in the home directory. Workspaces with externally killed zmx sessions remained in state and could not be restored.
- **Root cause:** Pane restoration treated a missing zmx session as a fatal shell exit, so Kitty could not reach attachment readiness and was killed by the launcher. The daemon trusted stale `BackendReady` state and never reconciled it with `zmx list`. Kitty's built-in same-CWD actions inspected the visible `zka -> zmx` proxy process, whose CWD was home, rather than using the shell's last OSC-7-reported directory.
- **Fix:** The daemon now reconciles authoritative panes against zmx every two seconds and before attach. A partial failure persists a dead-backend state and restores a ready static placeholder; Ctrl-C exits that placeholder so the existing watcher removes the pane. An all-dead workspace goes through the normal durable workspace deletion path. Managed Kitty overrides the three `*_with_cwd` action names to launch from `last_reported`, and pane allocation persists that inherited CWD locally and remotely. The canonical command is now `workspace attach`; `open` is intentionally rejected. Local/remote protocol version is 4 and package version is 0.4.0.
- **Evidence:** `go test ./...`, `go test -race ./...`, `go vet ./...`, and `git diff --check` pass. Kitty 0.47's live config parser resolves each managed alias to a `launch` action with the expected `--type` and `--cwd=last_reported` arguments.
- **Regression test:** `internal/zka/lifecycle_test.go` covers partial marking, all-dead deletion, startup grace, and stale unstarted cleanup. `internal/zka/cli_test.go` covers dead-pane text/Ctrl-C input, same-CWD aliases, and the attach-only CLI. `internal/zka/daemon_test.go` covers durable local/remote pane CWD. `internal/zka/remote_test.go` covers dead-placeholder readiness through origin heartbeats.
- **Related:** The earlier workspace lifecycle work already centralized pane/workspace removal and durable retry. This fix reuses that path; it does not add a second deletion mechanism. Attachment readiness now retains the semantic validation failure instead of masking it with a later socket error after Kitty exits.
- **Status:** DONE
